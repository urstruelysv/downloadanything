# Bug: All downloads routed through R2 regardless of size

**Category:** bug
**Status:** needs-triage

## Description

`runDownload()` in `worker/src/extractor.ts` never sets `sizeBytes` on the returned `DownloadResult`. In `worker/src/app.ts`, the condition `sizeBytes !== undefined && sizeBytes <= STREAM_LIMIT_BYTES` evaluates to `false` for every download, causing all files — even tiny ones — to be uploaded to Cloudflare R2 and served via signed URL.

This adds unnecessary latency and R2 storage/egress cost to every single download.

## Root cause

`runDownload()` spawns yt-dlp with `-o -` (stdout streaming). The file size is not available until the stream completes. The return value hardcodes `sizeBytes: undefined`.

## Files affected

- `worker/src/extractor.ts` — `runDownload()` (line ~192)
- `worker/src/app.ts` — size check condition (line ~58)

## Fix applied

Inverted the condition to stream by default. Only uploads to R2 when `sizeBytes` is explicitly known AND exceeds `STREAM_LIMIT_BYTES` (50MB). Unknown-size downloads now stream directly, which is correct for the vast majority of files.

```diff
- if (sizeBytes !== undefined && sizeBytes <= STREAM_LIMIT_BYTES) {
-   // stream
- } else {
-   // R2
- }
+ if (sizeBytes !== undefined && sizeBytes > STREAM_LIMIT_BYTES) {
+   // R2 (known large file)
+ } else {
+   // stream (unknown or small)
+ }
```

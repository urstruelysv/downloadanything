# Bug: Worker process crashes on yt-dlp spawn failure

**Category:** bug
**Status:** needs-triage

## Description

In `worker/src/extractor.ts`, `runDownload()` uses `throw` inside a Node.js event listener (`child.on("error", ...)`). A `throw` inside an event callback does not propagate to the caller — it becomes an unhandled exception that crashes the entire worker process.

If yt-dlp is not installed, the binary path is wrong, or the system is out of file descriptors, one failed download kills all in-flight requests.

## Root cause

```javascript
child.on("error", (e) => {
  throw new ExtractError("degraded", 502, e.message); // unhandled!
});
```

The function returns a result synchronously before the event loop processes the `error` event. The `throw` has no Promise or try/catch to catch it.

## Files affected

- `worker/src/extractor.ts` — `runDownload()` (line ~192)

## Fix applied

Rewrote `runDownload` to return a Promise. Resolves on `child.stdout` `readable` event (yt-dlp producing output). Rejects on `error` event (spawn failure) or non-zero `close` event (yt-dlp exited with error before producing output). Stderr is collected and included in the error message for diagnostics.
